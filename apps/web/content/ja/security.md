# セキュリティポリシー

securePDF は、利用者のファイルを不要に預からないことを基本方針にしています。

ブラウザ内で処理できる操作はローカルで完結させます。サーバー処理が必要な場合も、保存しない、内容をログに残さない、認証と制限をかける、という前提で扱います。

## ローカル処理の範囲

- PDF のページ編集、JPEG / PNG の追加、PDF 出力はブラウザ内で実行します。
- この範囲では、選択したファイル本文を securePDF のサーバーへ送信しません。
- 編集状態はブラウザのメモリ上で扱い、タブを閉じると保持されません。

## サーバー処理が必要な範囲

- docx / xlsx / pptx などの Office 変換は、変換品質と互換性のためバックエンドを利用します。
- 変換要求は Cloudflare Worker を経由し、設定済みの変換バックエンドへ送られます。
- 変換バックエンドは API キーと日次クレジットで利用を制限します。
- Cloudflare Worker 自体は重いファイル解析を行わず、必要な処理を分離された変換環境へ委譲します。

## ログと保存

- アップロードされたファイル本文を永続保存しないことを前提にします。
- ログにはファイル内容を記録せず、障害調査に必要なリクエスト ID や粗いメタデータに限定します。
- ファイル名も個人情報を含む可能性があるため、扱いを最小限にします。

## 入力防御

PDF や画像は信頼できない入力として扱います。

- ファイル形式、サイズ、ページ数、出力サイズの制限を設けます。
- URL 指定による取り込みは提供せず、SSRF につながる遠隔取得を避けます。
- `Content-Disposition` に入る出力ファイル名はサニタイズします。
- 暗号化 PDF、壊れた PDF、対応外形式は明示的なエラーとして扱います。

## 認証と制限

Office 変換などのバックエンド処理では、`X-API-Key` を使います。API キーは認証メニューまたは Google 認証導線で設定され、変換時に日次クレジットを消費します。

変換の乱用を避けるため、API キー検証、クレジット消費、レート制限、入力サイズ制限を組み合わせます。
