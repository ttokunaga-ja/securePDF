# セキュリティポリシー

securePDF は、利用者のファイルをできるだけ手元のブラウザ内で処理する方針です。

このページでは、どの操作でファイルが送信されるか、API キーがどこに保存されるか、エラーや制限が出たときに何を確認すればよいかを説明します。

## ファイルの扱い

| 操作 | ファイル送信 | 説明 |
| --- | --- | --- |
| PDF の並べ替え、削除、回転、反転 | 送信しません | ブラウザ内で処理します。 |
| JPEG / PNG の追加 | 送信しません | 画像をブラウザ内で PDF ページにします。 |
| 編集後 PDF のダウンロード | 送信しません | ブラウザ内で生成した PDF を保存します。 |
| docx / xlsx / pptx の PDF 変換 | 送信します | Office ファイルを PDF に変換するため、認証後に変換サーバーへ送信します。 |

PDF や画像だけを扱う場合、ファイル本文は securePDF のサーバーへ送られません。Office ファイルを変換する場合だけ、変換に必要なファイル本文が送信されます。

## Office 変換時に送信されるもの

Office 変換では、次の情報を送信します。

- 変換する Office ファイル。
- ファイル名とファイル形式。
- 認証に使う API キー。

変換結果の PDF はブラウザへ返され、ページとして取り込まれます。変換用に送信されたファイル本文を、利用者が再取得できる保存場所として保持しません。

## API キーの扱い

API キーは `tkp_` で始まる文字列です。右上の「その他」メニューから「認証」を開くと、API キーを入力できます。

入力した API キーや Google 認証で取得した API キーは、このブラウザの保存領域に保存されます。別のブラウザや別の端末には自動では引き継がれません。

Office 変換時は、その API キーを使って認証し、1 回あたり 5 クレジットを消費します。API キーが無効な場合やクレジットが不足している場合、変換は実行されません。

## ログについて

障害調査のため、リクエスト ID やエラー種別などの情報を扱う場合があります。ファイル本文をログに記録することは意図していません。

ファイル名には個人情報が含まれることがあるため、必要以上に扱わない方針です。

## 利用時の注意

- 機密性の高い Office ファイルを変換する場合は、ファイルが変換サーバーへ送信されることを確認してから利用してください。
- PDF や画像だけを編集する場合は、サーバー送信なしで利用できます。
- 共有端末では、ブラウザに保存された API キーが残る可能性があります。利用後はブラウザのサイトデータを削除するなど、端末の利用状況に合わせて管理してください。
- URL を指定して外部ファイルを取得する機能はありません。利用者が選択したローカルファイルだけを扱います。
